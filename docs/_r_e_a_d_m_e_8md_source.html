1 大変
2 ================================================================================
3 taiHEN is a CFW framework for PS Vita&trade;. When loaded with a kernel exploit,
4 it acts as a common substrate for patching the system. taiHEN provides three
5 main facilities:
6 
7 1. It disables code signature checks to allow unsigned executables.
8 2. It exposes kernel peek/poke syscalls to user applications and allows loading
9  of kernel modules.
10 3. Most importantly, it provides an API for hooking and replacing functions
11  based off of [substitute](http://github.com/comex/substitute).
12 
13 The last point means that developers can add custom patches to kernel, system
14 applications, and games alike.
15 
16 Building
17 --------------------------------------------------------------------------------
18 To build, you need the latest version of the
19 [toolchain](https://github.com/vitasdk/buildscripts) with kernel support. Then
20 just use CMake to build.
21 
22 ```bash
23 $ mkdir build && cd build
24 $ cmake ../
25 $ make
26 ```

28 Installation
29 --------------------------------------------------------------------------------
30 taiHEN requires a separate kernel exploit to run. Once the exploit loads
31 `taihen.skprx` to the kernel, taiHEN will take care of the rest. Please refer to
32 documentations for the exploit for more information.
33 
34 Plugins
35 --------------------------------------------------------------------------------
36 Plugins are loaded either into kernel after taiHEN is loaded or on demand when
37 an application is launched. taiHEN reads the configuration file in
38 `ux0:tai/config.txt`.
39 
40 The configuration that determines the plugins to load and the load order can
41 be found in `ux0:tai/config.txt`. The format is very simple and self
42 explanatory.
43 
44 ```text
45 # ignored line starting with #
46 # Kernel plugins are started with taiHEN and are in this section
47 *KERNEL
48 ux0:app/MLCL00001/henkaku.skprx
49 ux0:path/to/another.skprx
50 ux0:tai/plugin3.skprx
51 ux0:data/tai/plugin4.skprx
52 ux0:data/tai/plugin5.skprx
53 # titleid for SceSettings
54 *NPXS10015
55 ux0:app/MLCL00001/henkaku.suprx
56 ux0:data/tai/some_settings_plugin.suprx
57 # titleid for Package Installer
58 *NPXS10031
59 ux0:path/to/some_pkg_installer_plgin.suprx
60 # titleid for SceShell is special (does not follow the XXXXYYYYY format)
61 *main
62 ux0:app/MLCL00001/henkaku.skprx
63 ux0:data/tai/shell_plgin.skprx
64 ```
65 
66 The key things to note are
67 
68 1. `#` begins a comment, `*` begins a section, and any other character begins
69 a path.
70 2. `KERNEL` is a special section name denoting to load a kernel plugin when
71 taiHEN is started up. All other section names are the title id of the
72 application/game in which to load the plugin at startup. Note that SceShell
73 has a special title id of `main`.
74 3. In each section, there is a list of plugin paths that will be loaded in
75 order. Paths can be anywhere but it is recommended that plugins reside in
76 `ux0:tai` or `ux0:data/tai`. It is valid to have one plugin in multiple
77 sections but the developer must ensure that the plugin knows which application
78 it is loaded in if it needs to do things differently.

80 API
81 --------------------------------------------------------------------------------
82 taiHEN exports an API interface both to kernel and to user. This interface is
83 found in the [documentation pages](@ref taihen). You can either download the
84 release or build taiHEN yourself. After that, you can include `taihen.h` in your
85 project and link with `libtaihen_stub.a` and use `taihen.json` with vita-
86 toolchain.
